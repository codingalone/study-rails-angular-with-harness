# Gemini ドキュメント整合性レビュー

Gemini CLI を使ってリポジトリのドキュメント間の整合性レビューを実行します。

以下のコマンドを実行し、結果を解析して報告してください:

```bash
gemini -m gemini-2.5-pro --no-sandbox -p "あなたはドキュメントレビュアーです。このリポジトリの docs/ 配下の全ドキュメント（PROJECT_CHARTER.md, ARCHITECTURE.md, TESTING_STRATEGY.md, SECURITY_POLICY.md, INFRASTRUCTURE.md, CI_CD.md, DEPLOYMENT.md, DEVELOPMENT_GUIDE.md, SPECIFICATION_PROCESS.md）と CLAUDE.md の間で、以下の観点で整合性レビューを行ってください。

## レビュー観点
1. 用語の統一: 同じ概念に異なる用語が使われていないか
2. 数値・設定値の一貫性: カバレッジ基準、コスト見積もり、ポート番号、バージョン番号などが文書間で矛盾していないか
3. 参照の正確性: 他ドキュメントへの参照・リンクが正しいか
4. 役割・責任の定義の一貫性
5. フロー・プロセスの整合性
6. ディレクトリ構成の一貫性
7. セキュリティルールの一貫性
8. 技術スタック・バージョンの一貫性

ファイルをすべて読んで包括的にレビューしてください。修正は行わず、レビュー指摘のみ出力してください。

各指摘を以下の形式で出力:
### 指摘 N: タイトル
- 重要度: Critical / High / Medium / Low
- 関連ファイル: file1.md, file2.md
- 内容: 具体的な矛盾・不整合の説明
- 該当箇所: 各ファイルの該当セクション
- 推奨修正: どう修正すべきか"
```

結果を要約し、指摘事項のリストを整理して報告してください。
